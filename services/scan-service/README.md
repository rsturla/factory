# Scan Service

Runs vulnerability scans against container image SBOMs using Grype, persists findings, and serves results via a REST API. Automatically rescans images when the vulnerability database updates.

## Components

- **Scanner** — Workqueue reconciler that fetches SBOMs from blob storage, runs Grype against them, and stores vulnerability findings in the database.

- **DB Updater** — Keeps the Grype vulnerability database current. When a new database version is detected, enqueues rescans for all recent platforms to pick up newly disclosed vulnerabilities.

- **Syncer** — Periodic job that identifies platforms with SBOMs that haven't been scanned against the current vulnerability database and enqueues them for scanning.

- **API Server** — REST interface for querying scan results and findings by platform.

## Storage

- **PostgreSQL** — Scan records, vulnerability findings, and scanner database state.
- **Blob storage** — Reads SBOMs generated by the catalog service.
